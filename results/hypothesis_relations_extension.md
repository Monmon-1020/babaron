# プロトコル拡張: 仮説間関係の明示的分類

## 1. 問題の発見

Kelly & Sharot (2025) の論文を評価対象として提案手法（3層監督プロトコル）を実装・実行したところ、以下の不整合が観察された。

### 論文の主張

- H1: ウェブ閲覧コンテンツのvalenceは気分に因果的影響を与える（コンテンツ→気分）
- H2: 気分状態がその後の閲覧コンテンツの選択に因果的影響を与える（気分→コンテンツ）
- 論文の結論: **双方向の因果関係が成立**（H1とH2が共に支持される）

### プロトコルの出力

- H1: survive（正しい）
- H2: **reject**（誤り）
- AIはH2を棄却した。理由: H1の実験的確認（ネガティブコンテンツ→気分悪化）がH2の反証条件を発火させたと判断。

### 原因の特定

問題は特定のケースや特定のLLMの能力不足ではなく、**プロトコルの構造的制約**に起因していた。

第3章のStage 1は「競合仮説の列挙」を要求する。この「競合」という語は暗黙に仮説間の相互排他性を前提としており、以下の連鎖が生じた:

1. S1の「弁別予測（distinctive prediction）」の記述で、H1とH2が対立するかのような条件が設定された
2. S2の判定規則（accept/reject/hold）が、相手仮説の結果に依存する形で書かれた
3. S3でH1を支持するエビデンスが得られた時点で、H2のreject条件が自動的に満たされた

つまり、プロトコルが仮説間の関係を排他的と仮定する構造になっていたため、共存可能な仮説の一方が誤って棄却された。

## 2. 問題の一般性

この問題はKelly & Sharot論文に固有のものではない。観察的因果推論において、以下のような非排他的な仮説構造は頻出する。

| パターン | 例 | 排他前提の帰結 |
|---------|---|-------------|
| 双方向因果 | A→BかつB→A | 片方のacceptが他方のrejectを誘発 |
| 部分媒介 | AがMを通じてBに効くが直接効果もある | 媒介仮説と直接効果仮説の一方が棄却される |
| 複数メカニズムの並存 | 効果が経路1と経路2の両方で生じる | 経路1の確認が経路2の棄却に使われる |
| スケール依存的効果 | 短期では効果あり、長期ではなし | 時間スケールの異なる仮説が競合として扱われる |

これらのいずれにおいても、仮説間の関係が明示されなければ、S3の判定段階で誤った排除が生じうる。プロトコルが「何を監督するか」を定義する以上、仮説間の関係は監督対象に含まれるべき構成要素である。

## 3. 拡張の内容

### 3.1 S1への追加: hypothesis_relations フィールド

Stage 1の出力に、仮説ペアごとの関係タイプを宣言する構造を追加した。

```json
{
  "hypotheses": [ ... ],
  "hypothesis_relations": [
    {
      "pair": ["H1", "H2"],
      "relation": "independent",
      "note": "双方向因果として共存可能"
    },
    {
      "pair": ["H1", "H3"],
      "relation": "exclusive",
      "note": "H3はH1の効果が交絡と主張するため排他的"
    }
  ]
}
```

関係タイプは3種類:

- **exclusive（排他的）**: 一方が正しければ他方は棄却される。従来の「競合仮説」に相当。例: 「処置効果がある」vs「効果は交絡による見かけ」
- **independent（独立）**: 両方が同時に成立しうる。各仮説は自身のエビデンスのみで判定される。例: 双方向因果、並存メカニズム
- **nested（入れ子）**: 一方が他方の特殊ケースまたは部分集合。親仮説が棄却されれば子も棄却される。例: 「効果がある」（親）→「効果はメカニズムMを通じる」（子）

### 3.2 S2への波及: 判定規則の独立性制約

S1でindependentと宣言された仮説ペアについては、S2の判定規則（accept_if / reject_if / hold_if）が相手仮説の成否に依存してはならないという制約を導入した。

修正前（排他前提）:
```
H2の reject_if: "H1の実験操作で気分への因果効果が確認された場合"
```

修正後（独立制約）:
```
H2の reject_if: "気分を操作した後の自由閲覧で条件間のvalence差が非有意の場合"
```

これにより、各仮説の判定条件が自身のエビデンスのみに基づく形に矯正される。

### 3.3 S3への波及: 独立判定の許容

S3の結論導出において、independentな仮説が複数surviveすることを明示的に許容する。従来の「競合仮説の排除」は、exclusiveな仮説に対してのみ適用される。

結論の強さの段階づけも修正:
- strong: 識別仮定が支持され、**exclusiveな**競合仮説が排除され、ロバストネスが確認された
- weak/holdは変更なし

### 3.4 Layer 2への追加: ルーブリック項目

ルーブリック（Layer 2）に「仮説間の関係の明示」を評価項目として追加した。

| 評価 | 基準 |
|------|------|
| Explicit (2) | 全仮説ペアの関係が理由付きで明示されている |
| Partial (1) | 関係タイプは指定されているが理由が不十分 |
| Absent (0) | hypothesis_relationsが定義されていない |

## 4. 検証結果

同一のケース（Kelly & Sharot 2025）、同一のモデル（gpt-5.4-mini）で、拡張前後の出力を比較した。

### 拡張前

| 仮説 | 判定 | 論文との整合 |
|------|------|-----------|
| H1: コンテンツ→気分 | survive | ✅ |
| H2: 気分→コンテンツ | reject | ❌ |
| H3: 交絡 | reject | ✅ |

S1でH1とH2が排他的競合として構造化されたため、H1のacceptがH2のrejectを誘発した。

### 拡張後

| 仮説 | 判定 | 論文との整合 |
|------|------|-----------|
| H1: コンテンツ→気分 | survive | ✅ |
| H2: 気分→コンテンツ | survive | ✅ |
| H3: 交絡 | hold | ✅ |

S1でH1-H2がindependentと宣言されたことで、各仮説が自身のエビデンスで独立に判定され、双方向因果が正しく捕捉された。

### LLMの自律的判断

重要な点として、hypothesis_relationsの設定（exclusive/independent/nested）は、ケースデータや正解ラベルとしてあらかじめ与えたものではなく、**LLM（Designer）がS0の研究課題を読み取って自律的に出力した**ものである。プロトコルが「関係を宣言せよ」という構造的な枠を提供することで、LLMが適切な関係分類を行えるようになった。

## 5. 位置づけ

この拡張は、第3章3.2節（Protocol: 主張形成過程の構成要素）のStage 1「競合仮説の構造化」に対する追加的な構成要素である。

元のプロトコルは仮説間の関係を暗黙に排他的と仮定していた。この拡張は、仮説間の関係を明示的に宣言させることで:

1. **監督対象の拡大**: 仮説間の関係そのものが監査可能な構成要素となる
2. **誤った排除の防止**: independent仮説の判定条件が相互汚染しない構造を保証する
3. **プロトコルの適用範囲の拡大**: 双方向因果、部分媒介、複数メカニズムなど、排他的競合に収まらない因果構造を持つ研究にも適用可能になる

Layer 3（機械的チェック）の観点では、仮説間の関係タイプとS3の判定結果の整合性を決定論的に検査可能であり、LLMの内容判断に依存しない。したがって、この拡張はプロトコルの設計思想（何が機械的に検査可能で、何が内容判断を要するかの境界を明らかにする）と整合する。
