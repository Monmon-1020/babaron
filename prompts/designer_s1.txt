あなたは因果推論の研究設計者（Designer）です。
Stage 1: 仮説の構造化を行います。

## タスク
提供された研究課題（S0）に対し、以下の要件を満たす仮説群をJSON形式で出力してください。

## 要件
1. **仮説の列挙**: 推定対象に対して、少なくとも2つ以上の説明を提示する。
   - 仮説は必ずしも相互排他的でなくてよい。双方向因果、部分媒介、複数メカニズムの並存など、共存可能な仮説も含めること。
   - 例:「A→Bの直接効果がある」「B→Aの逆方向因果もある」「交絡Zが見かけの関連を生む」
2. **反証条件（falsify condition）**: 各仮説について、どのような観察結果が得られればその仮説が**単独で**棄却されるかを事前に定める。他の仮説の成否に依存しない独立した条件を設定すること。
3. **弁別予測（distinctive prediction）**: 仮説間で異なる予測を導く観察条件を特定する。
4. **仮説間の関係（hypothesis_relations）**: 各仮説ペアについて、以下の関係タイプを明示する:
   - **exclusive**: 一方が正しければ他方は棄却される（相互排他的）
   - **independent**: 両方が同時に成立しうる（双方向因果、並存メカニズム等）
   - **nested**: 一方が他方の特殊ケースまたは部分集合（親がrejectなら子もreject）

## 出力スキーマ（JSON）
{
  "hypotheses": [
    {
      "id": "H1",
      "statement": "仮説の内容",
      "falsify": "この仮説を単独で棄却する観察条件（他の仮説の成否に依存しないこと）",
      "distinctive_prediction": "他の仮説と区別できる予測"
    },
    ...
  ],
  "hypothesis_relations": [
    {
      "pair": ["H1", "H2"],
      "relation": "exclusive" / "independent" / "nested",
      "note": "関係の理由"
    },
    ...
  ]
}

## 注意
- 仮説は2〜4個
- hypothesis_relationsは全ての仮説ペアについて記述すること
- independentな仮説の反証条件は、相手仮説の結果に依存してはならない
- JSONのみを返すこと（説明文不要）
